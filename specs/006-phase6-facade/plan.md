# Implementation Plan: Phase 6 — Facade Unit Tests

**Branch**: `006-phase6-facade` | **Date**: 2026-04-02 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `specs/006-phase6-facade/spec.md`

## Summary

Implement five unit test classes covering the facade layer of the NexRAG RAG pipeline:
`IngestionFacadeImpl`, `CrudFacadeImpl`, `StreamingFacadeImpl`, `VoiceFacadeImpl`, and
`DuplicateChecker`. Each facade delegates to dependencies injected via constructor; all
dependencies are mocked with Mockito. The suite must reach ≥ 80 % line + branch coverage,
with 100 % branch coverage on `DuplicateChecker` (safety-critical deduplication path per
Constitution Principle IV).

## Technical Context

**Language/Version**: Java 21  
**Primary Dependencies**: JUnit 5 (Jupiter), Mockito, AssertJ, Spring Boot Test, reactor-test (StepVerifier for Flux-based streaming)  
**Storage**: N/A — all dependencies mocked; no real infrastructure in unit tests  
**Testing**: JUnit 5 + `@ExtendWith(MockitoExtension.class)`, AssertJ, `StepVerifier` for reactive streams  
**Target Platform**: Maven Surefire (Spring Boot 3.4.2 BOM)  
**Project Type**: Unit test suite — service facade layer  
**Performance Goals**: Each test method < 500 ms (Constitution Principle I)  
**Constraints**: No real network calls; no file-system I/O; every dependency injected via `@Mock` / `@InjectMocks`  
**Scale/Scope**: 5 `*Spec.java` classes, ~18–22 `@Test` methods total
