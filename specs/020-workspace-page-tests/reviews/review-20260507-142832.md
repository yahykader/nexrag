## Integration Test Logic Verification

The critical path for T011 was verified against `toast-container.component.html`:

1. `beforeEach`: `toastsSubject = new Subject<Toast>()` → `createComponent()` → `ToastContainerComponent.ngOnInit()` subscribes to `toastsSubject.asObservable()` via the lazy getter.
2. Test body: `toastsSubject.next({ id: '1', type: 'success', ... })` → subscription fires synchronously → `this.toasts.push(toast)`.
3. `spectator.detectChanges()` → Angular renders `*ngFor="let toast of toasts"` → the div receives `class="toast show bg-success text-white"`.
4. `expect(spectator.query('.toast.bg-success')).not.toBeNull()` → ✅ selector matches.

The error test follows the same path with `type: 'error'` → `[class.bg-danger]="toast.type === 'error'"` → `.toast.bg-danger` ✅.

No `duration` is set in either emission, so no `setTimeout`-triggered removal fires. Assertion timing is safe.
