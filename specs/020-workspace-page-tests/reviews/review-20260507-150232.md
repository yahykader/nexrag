## T012 Fix Verification

The definitive T012 body:

```ts
it('[INTÉGRATION] doit être compatible avec le store complet (5 slices)', () => {
  const updatedState = {
    ...FULL_STATE(),
    ingestion: { ...FULL_STATE().ingestion, activeUploads: 1 },
  };
  store.setState(updatedState);
  spectator.detectChanges();
  expect(spectator.component).toBeTruthy();
});
```

**Why this is correct**: `WorkspaceComponent` is a pure layout shell — it declares no NgRx selectors and does not subscribe to the store. The maximum meaningful assertion for SC-002 at this level is that a real state mutation across all 5 slices does not destabilize the component's mounted state. `store.setState()` delivers the full updated state object to the mock store; `detectChanges()` propagates any potential store-driven updates through the Angular change detection tree; `expect(spectator.component).toBeTruthy()` confirms the component remains mounted and error-free after the state transition. This satisfies FR-007 and SC-002 for a shell with no direct store subscriptions.
